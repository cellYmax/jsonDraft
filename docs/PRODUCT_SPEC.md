# Product Spec

最后更新：2026-05-22

## 产品定位

jsonDraft 是一个本地桌面 JSON 编辑器，面向需要快速查看、校验、整理、压缩和转义 JSON/JSONC 文本的开发者。v1 优先保证单文件编辑体验清晰可靠，不追求多标签、Schema 平台或大型文件处理。

## v1 目标

- 打开、编辑、保存、另存为本地 `.json` 和 `.jsonc` 文件。
- 默认启动时加载项目内置示例文件，方便第一次打开即可验证功能。
- 支持严格 JSON 与 JSONC 两种模式切换。
- 在编辑时实时展示解析状态、错误位置、根节点类型、条目数量、格式状态、当前 JSON Path 和树形导航。
- 支持格式化、压缩、压缩并转义、去除转义。
- 手动保存为主，关闭或切换文件前保护未保存内容。
- 文件大小目标为 10MB 以下。

## 非目标

- 不做多标签页。
- 不做目录浏览器。
- 不做 Schema 校验和远程 Schema 获取。
- 不做 diff、历史版本、插件系统。
- 不做自动保存。
- 不做超过 10MB 文件的大文件优化。

## 主要用户流程

### 启动

1. 应用启动后默认显示 `examples/customer-profile.json` 内容。
2. 文件状态为未绑定本地路径的干净草稿。
3. 底部状态栏显示文件名、保存状态、光标位置、文件大小和校验状态。

验收标准：

- 首屏不是空白编辑器。
- 示例内容可被格式化、压缩、树形导航和路径面板识别。

### 新建文件

1. 用户点击“新建”或使用 `Cmd/Ctrl+N`。
2. 如果当前文件有未保存修改，需要二次确认。
3. 新文件内容为基础 JSON 模板，名称为 `未命名.json`，路径为空。

验收标准：

- 新建后 `dirty=false`。
- 保存时若无路径，应走“另存为”流程。

### 打开文件

1. 用户点击“打开”或使用 `Cmd/Ctrl+O`。
2. Tauri 后端弹出文件选择框。
3. 仅允许选择 JSON、JSONC 或全部文件。
4. 后端拒绝超过 10MB 的文件。
5. 打开成功后更新文件状态、最近文件和光标位置。

验收标准：

- 用户取消选择时不报错，只显示“操作已取消”。
- 打开失败时保留原文件内容。
- 最近文件最多保留 5 条。

### 保存与另存为

1. 用户点击“保存”或使用 `Cmd/Ctrl+S`。
2. 如果文件已有路径，直接写回该路径。
3. 如果文件无路径，转为“另存为”流程。
4. 用户点击“另存为”或使用 `Cmd/Ctrl+Shift+S` 时，弹出保存对话框。

验收标准：

- 保存成功后 `dirty=false`。
- 保存期间如果内容又被编辑，保存结果只更新路径和已保存基线，不覆盖当前编辑内容。
- JSONC 模式保存时原样保存编辑器内容，包括注释和尾逗号。

### 模式切换

1. 用户通过右上角 JSON/JSONC 分段按钮切换模式。
2. 严格 JSON 模式下，注释和尾逗号视为错误。
3. JSONC 模式下，允许注释和尾逗号。

验收标准：

- Monaco 诊断选项与当前模式一致。
- 解析模块输出与模式一致。
- 切换模式不改写文件内容。

### 格式化

1. 用户点击“格式化”或使用 `Cmd/Ctrl+Enter`。
2. 当前内容必须能按当前模式解析。
3. 严格 JSON 模式输出标准 JSON，两空格缩进，末尾换行。
4. JSONC 模式使用 `jsonc-parser` 的格式化能力，尽量保留注释。

验收标准：

- 无效内容不能格式化。
- 格式化后文件进入正确的 dirty 状态。

### 压缩

1. 用户点击“压缩”或使用 `Cmd/Ctrl+Shift+M`。
2. 当前内容必须能按当前模式解析。
3. 输出为标准 JSON 单行字符串。

验收标准：

- JSONC 输入压缩后会丢弃注释并输出标准 JSON。
- 压缩结果不包含多余空白。

### 压缩并转义

1. 用户点击“压缩转义”或使用 `Cmd/Ctrl+Shift+E`。
2. 先压缩为标准 JSON，再包装成 JSON 字符串。

验收标准：

- 输入 `{"a":1}` 输出 `"{\"a\":1}"` 形式。
- JSONC 输入按 JSONC 解析，但输出仍是标准 JSON 字符串。

### 去除转义

1. 用户点击“去转义”或使用 `Cmd/Ctrl+Shift+U`。
2. 支持带外层引号的 JSON 字符串，也支持常见的无外层引号粘贴文本。
3. 解码后内容会被格式化。

验收标准：

- `"{\"a\":1}"` 可以还原为格式化 JSON。
- 非 JSON 文本应显示明确错误。

### 错误定位与结构查看

1. 右侧错误面板列出解析错误。
2. 点击错误可跳转到对应行列。
3. 结构面板展示根类型、条目数、排版和大小。
4. 树形导航展示最多 250 个节点，可收缩。
5. 点击树节点跳转到对应 offset。
6. 当前位置面板显示当前光标 JSON Path，可复制。

验收标准：

- 树形导航内容不应撑破侧栏。
- 当前节点在树中高亮。
- 无效 JSON 不展示树节点。

## UI 布局

- 顶部工具栏：品牌、文件操作、转换操作、模式切换。
- 中部工作区：左侧 Monaco 编辑器，右侧信息侧栏。
- 右侧侧栏：校验健康状态、错误、结构、树形导航、当前位置、快速复制、最近文件。
- 底部状态栏：dirty 点、文件名、保存状态、光标位置、大小、有效性、通知。

## 文案约定

- v1 默认中文 UI。
- 错误文案尽量可操作，不直接暴露底层英文异常。
- 用户主动取消的操作使用“操作已取消”，不视为失败。
